Binary Search Trees and Heaps

Binary Search Tree

The binary search tree additionally satisfies the bst property, which states that the key in each node must be greater than or equal to any key sotred in the left sub-tree, and less than or equal to any key stored in the right sub tree.

If its greater than, go right, if its less than, go left

BST's are very quick, but must be balanced. Maximum difference of nodes should be 1, anymore and it loses performance.

This could be really good for a JS dictionary(a dictionary of objects)

This would be really good to iterate and sort through

This is used more than linked lists

BST operations

  • insert, inserts a new node into the tree
  • find, finds a key in the tree and returns the data in the node
  • remove, removes a node given a key
  • size, gives the number of elements
  • print, returns a string representation of the tree structure

Binary Heap

Heaps are similar except the root element is the largest one

In computer science, a heap is a tree data structure which satisfies the heap property: which says that a parent node has to be greater than or equal to its children. the most common implementation is a binary heap in which the tree is a binary tree

The advantages of using a binary heap as a data structure: finding the max element is constant time, could be very space efficient, they are easy to implement

The disadvantages of using a Binary Heap as a data structure: slower than a stack, memory allocation is more complex

Applications: heapsort, Priority queue, Dijkstra's Shortest Path

Binary Heap Operations

  • insert, inserts a new node into the tree
  • find, fins a key in the tree and returns the data in the node
  • remove, removes a node given a key
  • extractMax, removes and returns the root node and rearranges the rest of the tree to restore the heap property
